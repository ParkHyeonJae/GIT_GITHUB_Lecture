  1. 1. Git & Github는 도대체 왜 배우는 거야? 🤔

    2. ## 배우는 이유는?

    1. 1. 실수했거나, 기획이 변경되더라도 GIT을 통해 특정한 과거의 파일로 돌아갈 수 있다.
    2. - 즉 conflict(내가 쓴 파일의 내용이 다른 사람의 코드 혹은 올린 코드와 겹쳤을때 뜨는 충돌 메시지) 와
      같은 기능 덕분에
      문제의 원인을 빠른 시간 내로 해결이 가능하다.
    3. 2. GIT을 이용해서 로컬(내 컴퓨터)에 작업한 내용들을 원격저장소(Github, Bitbucket, GitLab)에
      올려서 다른 사람들과 공유하거나 다른 컴퓨터에서 작업이 가능하다.
    4. 3. Action 또는 Branch 를 이용한 손쉬운 개발 환경 !
    5. - ex) Branch를 IOS, Android 두가지 버전으로 나눠서 어플리케이션을 개발할 수 있다.
    6. - ex) Action 기능을 이용한 Build & Test 자동화가능 (Circle-CI 대체)
    7. 4. PR(Pull Request)를 이용해서 프로젝트 이외의 사람이 프로젝트 내용에 간섭이 가능하다!
    8. - 물론 주인이 Request를 수락하면 Merge가 된다.

  3. 3. 마크다운(Markdown)을 활용한 나만의 회의록 만들기 📝

    4. ## 마크다운이란 뭘까?

    1. 마크다운은 웹상에서 글을쓰는 사람들을 위한 글쓰기 양식
    2. "#"의 개수에 따라서 제목크기가 달라진다.-> HTML의 h1, h2, h3 와 같은 개념 ("#"이 많을수록 더 작아짐)
    3. 인용을 하려면 ">"기호를 사용한다.
    4. 테이블은 "| Content |" 형식으로 제작이 가능하다 (가운데 정렬은 "|:----:|")
    5. 굵게 표현을 하려면 "**Content**"
    6. 기울어진 표현을 하려면 "_Content_"
    7. 굵고 기울어진 표현을 하려면 "**Content**" (또는 "___Content___")
    8. 링크 삽입방법 -> [1]: https://www.google.co.kr "구글" 선언 후 [구글로 이동][1] 이런식으로 사용가능하다.( 나무위키의 주석처럼 생각해면 쉽다 )

  4. 4. Git을 직접 이해하고 써보자 🚀

    5. ## Git의 요소

    1. GIT : 로컬에서 실행되는 관리 도구 원격 저장소(Github, BitBucket, GitLab): GIT의 내용을 올릴 수 있는 저장소
    2. GIT CLI : 명령어 형태로 GIT을 다룰 수 있다.(GIT의 모든 기능을 사용 할 수 있다.-> 우클릭 후 git bash 를 누르면 된다. cmd, PowerShall, Terminal에서도 사용가능)
    3. GIT GUI : GIT의 일부기능을 시각적으로 표현한 프로그램(Source Tree, Github Desktop 등등..)
    4. CLI를 통해 GIT의 원리를 이해하면 다른 모든 GUI 프로그램을 다룰 수 있어지기 때문에 어렵더라도 CLI 기준으로 교육을 진행했다. (GUI Functions ⊂ CLI Functions)
    5. 0.원하는 원격저장소(Github, Bitbucket, Gitlab)에서 자신만의 Repository(저장소)를 만들어준다.(위 강의에서는 Github로 진행헀다)
    6. 1.CLI 창을 띄운 뒤 "cd"라는 명령어를 이용해서 내가 작업하고자 하는 폴더로 이동한다. GIt-Practice라는 폴더를 만들었을때 -> cd Git-Practice
    7. 2.원하는 폴더로 이동했으면 "git init"이라는 명령어로 내가 앞으로 작업하게 될 공간을 지정해준다.
    8. 3.지정이 완료됬으면 내가 생성하고자 하는 파일을 만든다.(위 강의의 목적은 마크다운을 이용한 회의록 작성이므로 회의록을 만들었음)
    9. 4.파일을 생성했으면 "git status"를 입력해본다. -> 그러면 창에 Untracked ~ ... 라는 말이 뜨면 정상이다.(파일이 추적(파일관리)되지 않았다는 뜻)
    10. 5.Untracked 된 파일을 git add 을 입력하거나 "git add ." 또는 "git add *.Tracked시킬 파일확장자 ex) *.c"를 입력해서 Tracked(추적)시킨다
    11. 6."git commit -m "입력할 커밋 메시지" "를 입력해서 키밋을 등록 시켜준다.
    12. 7.원격 저장소 연결을 위해 "git remote add origin <내 Repository주소>"를 입력해준다.
    13. 8.잘 연결되었는지 확인하기 위해서 "git remote show origin"이라고 입력해본다. (입력 후 등록한 Repository 링크가 뜬다면 성공이다.)
    14. 9.이제 원격저장소로 작업한 것들을 보내기 위해서 "git push origin master"을 입력해준다.
    15. 10. Repository에 들어간다음 내가 추가한 파일이 올라가 있으면 성공이다.

  5. 5. 깃(Git)가 막힌(?) Git에 대한 팁 ! 🔍
